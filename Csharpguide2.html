Arrays and Collections

Arrays let you store multiple values in a single variable. Collections like List are more flexible because they can grow and shrink.

int[] scores = { 80, 90, 75 };
Console.WriteLine(scores[0]);

var names = new List { "Alice", "Bob", "Charlie" };
names.Add("Dana");
Console.WriteLine(names.Count);

Key highlights:

  • scores[0] gets the first item in the array.
  • List<string> stores a resizable list of strings.
  • Add() inserts new values at the end of the list.

Classes and Objects

Classes define blueprints for objects. An object is an instance of a class that holds data and behavior.

public class Student {
    public string Name { get; set; }
    public int Grade { get; set; }

    public void PrintInfo() {
        Console.WriteLine($"Name: {Name}, Grade: {Grade}");
    }
}

var student = new Student {
    Name = "Alice",
    Grade = 10
};
student.PrintInfo();

Key highlights:

  • public class Student creates a class named Student.
  • Properties store values inside the object.
  • PrintInfo() is a method that displays object data.

String Interpolation

String interpolation makes it easier to build text using variables and expressions.

string userName = "Alice";
int age = 21;
Console.WriteLine($"{userName} is {age} years old.");

Key highlights:

  • Start the string with $ to enable interpolation.
  • Use curly braces { } around variables and expressions.
  • Interpolation keeps strings readable and avoids lengthy concatenation.

Exception Handling

Exceptions let your program catch and respond to runtime errors instead of crashing.

try {
    int value = int.Parse("abc");
    Console.WriteLine(value);
} catch (FormatException) {
    Console.WriteLine("Please enter a valid number.");
} finally {
    Console.WriteLine("Done checking the input.");
}

Key highlights:

  • try runs the code that may throw an error.
  • catch handles specific error types.
  • finally always runs, even when an error occurs.
