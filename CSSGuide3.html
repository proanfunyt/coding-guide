CSS Grid Layout

CSS Grid gives you powerful control over two-dimensional layouts. Use rows and columns to place content in a grid.

.grid {
    display: grid;
    grid-template-columns: 1fr 2fr;
    grid-template-rows: auto auto;
    gap: 20px;
}

.header {
    grid-column: 1 / -1;
}

How it works:

  • display: grid makes the parent a grid container.
  • grid-template-columns defines the number and size of columns.
  • grid-column: 1 / -1 makes the element span the full width of the grid.

Advanced Selectors

Advanced selectors help you target elements with precision, even without extra classes or IDs.

article > p {
    margin-bottom: 16px;
}

nav a:first-child {
    font-weight: bold;
}

button:not(.primary) {
    opacity: 0.8;
}

How it works:

  • > targets direct children only.
  • :first-child selects the first matching item inside a parent.
  • :not() selects elements that do not match a selector.

Functions and Calculations

CSS offers functions like calc(), min(), max(), and clamp() to build flexible, responsive values.

.card {
    width: calc(100% - 40px);
    font-size: clamp(1rem, 2vw, 1.25rem);
}

.section {
    gap: min(2rem, 5vw);
}

How it works:

  • calc() performs math with lengths and percentages.
  • clamp() sets a value with a minimum, preferred, and maximum size.
  • min() and max() choose the smallest or largest value.

Custom Properties in Depth

Custom properties let you create theme variables that can be reused and updated across your stylesheet.

:root {
    --main-bg: #0f172a;
    --surface: #1e293b;
    --accent: #38bdf8;
    --radius: 12px;
}

.card {
    background: var(--surface);
    border-radius: var(--radius);
    box-shadow: 0 10px 30px rgba(0, 0, 0, 0.2);
}

.card:hover {
    --accent: #7dd3fc;
    border-color: var(--accent);
}

How it works:

  • Custom properties can be declared globally or on specific elements.
  • They can also change inside hover or media queries for dynamic theming.
  • Use var(--name, fallback) to provide a fallback value.

Layering and Z-Index

Control which elements appear on top using stacking order and positioning.

.modal {
    position: fixed;
    z-index: 50;
}

.overlay {
    position: fixed;
    z-index: 40;
}

How it works:

  • position creates a stacking context for z-index.
  • A larger z-index value places the element above smaller values.
  • Use position: relative to create local stacking contexts for nested elements.

Accessibility and Focus Styles

Good CSS supports keyboard users by making interactive elements visible when focused.

button:focus-visible {
    outline: 3px solid #38bdf8;
    outline-offset: 4px;
}

a:focus-visible {
    text-decoration: underline;
}

How it works:

  • :focus-visible applies styles only when an element is focused by keyboard or assistive tools.
  • Clear focus styles help users know where they are on the page.
  • Accessible design is essential for real-world websites.
